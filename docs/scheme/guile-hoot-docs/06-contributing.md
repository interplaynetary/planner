# Contributing to Guile Hoot

## Reporting Issues and Submitting Changes

The project welcomes community participation in two primary ways:

**Bug Reports:** Users who identify defects are encouraged to report them to the maintainers.

**Improvements:** Contributors with enhancements or fixes may submit their work for review.

## Process

Both issues and code contributions are managed through the project's version control platform. Issues can be filed and pull requests can be submitted on [GitLab](https://gitlab.com/spritely/guile-hoot).

Contributors should use GitLab as the central hub for all contribution-related activities, whether reporting problems or proposing code changes.
